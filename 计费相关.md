1. 基础货币
本平台采用 $ 作为站内基础计费单位。

充值规则
用户充值时，按照以下比例换算：

1 CNY = 1 $
例如：

充值 10 CNY，到账 10 $
充值 100 CNY，到账 100 $
2. API 计费方式
平台 API 调用费用基于 模型官方 API 单价 进行计算，并在此基础上乘以 分组倍率，得出平台最终扣费金额。

3. 计费公式
3.1 官方 API 费用
模型官方 API 费用计算公式如下：

```
官方费用 =
(输入 token 数 × 输入单价)
+ (输出 token 数 × 输出单价)
+ (缓存创建 token 数 × 缓存创建单价)
+ (缓存读取 token 数 × 缓存读取单价)
```

3.2 平台最终扣费
平台最终扣费公式如下：

> 平台扣费 = 官方费用 × 分组倍率
即：

先根据模型官方单价和实际 token 消耗计算官方费用
再乘以对应分组倍率
从账户余额中扣除最终金额
4. 分组倍率说明
平台可针对不同模型分组或渠道分组设置不同倍率。

分组倍率	平台扣费含义	相当于官方原价的倍率
| 分组倍率 | 平台扣费含义 | 相当于官方原价的倍率 |
| :---: | :--- | :---: |
| 1 | 按官方 USD 费用 1 倍扣费 | 约等于官方原价的 0.14 倍 |
| 1.2 | 按官方 USD 费用 1.2 倍扣费 | 约等于官方原价的 0.17 倍 |
| 2.5 | 按官方 USD 费用 2.5 倍扣费 | 约等于官方原价的 0.36 倍 |
5. 扣费规则说明
所有 API 调用均按实际消耗的 token 数量计费
不同模型的输入、输出、缓存创建、缓存读取单价可能不同
不同分组的倍率可能不同
若某模型不涉及某项计费，则该项费用按 0 计算
最终扣费金额以平台实际计算结果为准
6. 余额说明
API 请求产生费用后，系统将自动从账户余额中扣除对应的 $ 金额。

若账户余额不足，可能会出现以下情况：

请求被拒绝
请求执行失败
系统提示余额不足
具体以平台实际策略为准。

7. 计费总结
平台计费逻辑可总结为：

> 平台扣费 = 官方 API 费用 × 分组倍率

其中：
> 官方 API 费用 = 输入费用 + 输出费用 + 缓存创建费用 + 缓存读取费用

充值规则为：

1 CNY = 1 $
